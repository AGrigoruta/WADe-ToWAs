Login Page

The landing page is the dashboard which we will see later in this User Guide. If the user is not logged in, he will be redirected automatically to the login page. In the login page, the user has the possibility to use the Facebook account, in order for him to save some precious time, not to create a new account and customize it yet again. When he clicks on the login button, he will be presented with a dialog in which he will have to allow ToWAs to use some of the data Facebook provides (name and profile picture). If the login has been successful, he will be redirected to the landing page

User Details Prompt

The first thing he will see in the landing page is a form in which he will provide some basic data, for ToWAs to work. These questions include age range (18-24, 25-35, 36-50, 50+), whether his next trip is a Business Trip, Solo Trip, Family Trip or Friends Trip, whether he already has accommodation, whether he is a sports person, cultural person or in pilgrimage.

After he has completed all the needed details, he can proceed to use ToWAs at it's fullest potential.

Dashboard

This is the main dashboard. Here, the user can surf on the map however he wishes, although, the first time he enters, he will be centered to his current location (if he allows the Geolocation API to retrieve this data). From here, he can either refill the previous form, search for a trip destination to get itineraries or logout

Itineraries

When the user types a destination in the search bar at the top, he will retrieve, faster or slower, depending on his internet speed, a list of points of interest. Most of the results are either Museums, Hotels, Parks or Churches, data retrieved from DBPedia.

If he clicks on a selected marker, he will see more information about that specific point of interest. The details include the name, the ontology url for linked data and a short or long description about that specific place. When he decides to leave, the user can click on the greeting from the navbar (including the profile picture), and proceed to logout, having to log back in when he comes back again. If he doesn't want to do that, if he closes the tab or browser, the Facebook id remains active for some time
